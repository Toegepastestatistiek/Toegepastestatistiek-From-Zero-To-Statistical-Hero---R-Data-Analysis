Modus Berekenen in R

R heeft geen ingebouwde modus-functie. Hier leer je drie methoden om de modus te berekenen: via frequentietabellen, een eigen functie, en met tidyverse.

Waarom heeft R geen mode() functie?

Let op! De functie mode() in R geeft het datatype terug (numeric, character, etc.), NIET de statistische modus!

‚ö†Ô∏è Veelgemaakte fout R 
# Dit is FOUT - mode() geeft het datatype!
mode(c(1, 2, 2, 3))
[1] "numeric" ‚Üê Dit is het datatype, NIET de modus!

Methode 1: Frequentietabel (table)

De eenvoudigste methode is een frequentietabel maken en de hoogste frequentie aflezen:

modus-frequentietabel R 
# Stap 1: Maak frequentietabel
freq_table <- table(data$variabele)
print(freq_table)

# Stap 2: Vind de modus (waarde met hoogste frequentie)
modus <- names(freq_table)[which.max(freq_table)]
print(paste("Modus:", modus))

Voorbeeld: Relatiestatus

voorbeeld-relatiestatus R 
table(wave1_coded$W1_Relatiestatus)
Nee Ja 87 218

Interpretatie van de Output

Modus = "Ja" (hoogste frequentie: 218 respondenten)

  • 218 van de 305 respondenten (71.5%) hebben een relatie
  • 87 respondenten (28.5%) hebben geen relatie
  • Conclusie: De meerderheid van de steekproef heeft een relatie

Methode 2: Eigen Modus Functie

Maak een herbruikbare functie voor de modus:

modus-functie R 
# =============================================
# Eigen modus functie - herbruikbaar
# =============================================

calculate_mode <- function(x) {
  # Stap 1: Verwijder missing values (NA's)
  x <- x[!is.na(x)]
  
  # Stap 2: Maak frequentietabel
  freq_table <- table(x)
  
  # Stap 3: Vind modus (waarde met hoogste frequentie)
  mode_value <- names(freq_table)[which.max(freq_table)]
  mode_freq <- max(freq_table)
  
  # Stap 4: Geef resultaat terug als lijst
  return(list(
    mode = mode_value,
    frequency = mode_freq
  ))
}

# Gebruik de functie
result <- calculate_mode(wave1_coded$W1_Relatiestatus)
print(paste("Modus:", result$mode, "- Frequentie:", result$frequency))
[1] "Modus: Ja - Frequentie: 218"

Uitleg van de Functie

  • x[!is.na(x)] - Filtert alle NA (missing) waarden uit
  • table(x) - Telt hoe vaak elke waarde voorkomt
  • which.max(freq_table) - Vindt de positie van de hoogste waarde
  • names(freq_table)[...] - Haalt de naam (label) op van die positie

Uitgebreide Functie (met multimodale detectie)

modus-uitgebreid R 
# =============================================
# Uitgebreide modus functie
# Detecteert ook bimodale/multimodale verdelingen
# =============================================

calculate_mode_extended <- function(x) {
  # Verwijder NA's
  x <- x[!is.na(x)]
  freq_table <- table(x)
  max_freq <- max(freq_table)
  
  # Vind ALLE waarden met maximale frequentie
  modes <- names(freq_table)[freq_table == max_freq]
  
  # Check type verdeling
  if (length(modes) == length(freq_table)) {
    return("Geen modus (uniforme verdeling)")
  } else if (length(modes) > 1) {
    return(paste("Multimodaal:", paste(modes, collapse = ", ")))
  } else {
    return(paste("Modus:", modes))
  }
}

Wanneer gebruik je deze uitgebreide functie?

  • Bimodaal: Twee waarden komen even vaak voor (bijv. leeftijd met pieken bij 25 en 55 jaar)
  • Multimodaal: Meer dan twee waarden komen even vaak voor
  • Uniform: Alle waarden komen even vaak voor ‚Üí geen modus

Methode 3: Tidyverse Aanpak

modus-tidyverse R 
library(dplyr)

# Modus berekenen met tidyverse pipe syntax
wave1_coded |>
  filter(!is.na(W1_Relatiestatus)) |>   # Verwijder NA's
  count(W1_Relatiestatus, name = "frequentie") |>  # Tel frequenties
  mutate(percentage = round(frequentie / sum(frequentie) * 100, 1)) |>  # Bereken %
  arrange(desc(frequentie)) |>  # Sorteer aflopend
  slice(1)  # Eerste rij = modus
# A tibble: 1 √ó 3 W1_Relatiestatus frequentie percentage <fct> <int> <dbl> 1 Ja 218 71.5

Voordelen van Tidyverse Aanpak

  • Leesbaar: Code leest als een verhaal van links naar rechts
  • Flexibel: Makkelijk uit te breiden met extra berekeningen
  • Percentage: Direct inzicht in relatieve frequentie (71.5%)

Modus per Groep Berekenen

modus-per-groep R 
# Modus berekenen per subgroep (bijv. per geslacht)
wave1_coded |>
  group_by(W1_Geslacht) |>          # Groepeer op geslacht
  count(W1_Relatiestatus) |>        # Tel per groep
  slice_max(n, n = 1) |>           # Neem hoogste n per groep
  ungroup()
# A tibble: 2 √ó 3 W1_Geslacht W1_Relatiestatus n <fct> <fct> <int> 1 Man Ja 98 2 Vrouw Ja 120

Interpretatie

Bij beide geslachten is "Ja" de modus:

  • Mannen: 98 van hen hebben een relatie (modus)
  • Vrouwen: 120 van hen hebben een relatie (modus)

‚Üí Geen verschil in modus tussen groepen, maar wel in absolute aantallen

Belangrijke Tips

  • Gebruik altijd na.rm = TRUE of filter NA's eerst
  • Check voor multimodale verdelingen
  • De modus werkt op ALLE meetniveaus (nominaal, ordinaal, metrisch)
  • Bij continue data: overweeg categorisatie of gebruik andere maten
