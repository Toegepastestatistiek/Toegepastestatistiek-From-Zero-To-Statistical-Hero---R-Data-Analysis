Betrouwbaarheidsintervallen: Van Steekproef naar Populatie

Een betrouwbaarheidsinterval (BI) geeft een bereik waarbinnen de populatieparameter waarschijnlijk ligt, gebaseerd op je steekproef.

Het Concept van Betrouwbaarheidsintervallen

Wat is een 95% Betrouwbaarheidsinterval?

Correcte interpretatie:

"Als we oneindig vaak een steekproef nemen en telkens een 95% BI berekenen, zal 95% van deze intervallen de werkelijke populatiewaarde bevatten."

Voor één specifiek interval:

"We zijn 95% confident dat dit interval de populatieparameter bevat."

⚠️ FOUTE Interpretaties

  • ❌ "Er is 95% kans dat de populatiewaarde in dit interval ligt"
  • ❌ "95% van de data ligt in dit interval"
  • ❌ "We zijn 95% zeker dat..."

Waarom fout? De populatieparameter is een vaste waarde (geen kansverdeling). Het interval vángt deze waarde, of niet - er is geen "kans".

Berekening van een 95% BI voor het Gemiddelde

Formule:

$$ \text{95% BI} = \bar{x} \pm t_{\alpha/2, df} \times SE $$

Waarbij:

  • $\bar{x}$ = steekproefgemiddelde
  • $t_{\alpha/2, df}$ = kritieke t-waarde (tweezijdig, α=0.05)
  • $df$ = vrijheidsgraden (n - 1)
  • $SE$ = standaardfout = $\frac{SD}{\sqrt{n}}$

Stap-voor-Stap Voorbeeld

Gegeven: Relatiestress scores (schaal 1-5)

  • n = 150 respondenten
  • M = 3.45
  • SD = 0.82

Stap 1: Bereken SE

$$ SE = \frac{SD}{\sqrt{n}} = \frac{0.82}{\sqrt{150}} = \frac{0.82}{12.247} = 0.067 $$

Stap 2: Bepaal t-kritiek

Voor df = 150 - 1 = 149 en α = 0.05 (tweezijdig): t = 1.976

Stap 3: Bereken Foutenmarge

$$ \text{Foutenmarge} = t \times SE = 1.976 \times 0.067 = 0.132 $$

Stap 4: Bereken 95% BI

$$ \text{Ondergrens} = 3.45 - 0.132 = 3.318 $$

$$ \text{Bovengrens} = 3.45 + 0.132 = 3.582 $$

Resultaat:

95% BI = [3.32, 3.58]

Interpretatie: We zijn 95% confident dat het gemiddelde relatiestress niveau in de populatie tussen 3.32 en 3.58 ligt.

R Code voor Betrouwbaarheidsintervallen

Methode 1: Handmatig Berekenen

# Data voorbereiden
stress <- data_wave1$W1_Relstress1_Gem
n <- sum(!is.na(stress))
gemiddelde <- mean(stress, na.rm = TRUE)
sd <- sd(stress, na.rm = TRUE)

# SE berekenen
se <- sd / sqrt(n)

# t-kritiek vinden (95% BI, tweezijdig)
t_kritiek <- qt(0.975, df = n - 1)  # 0.975 = 1 - 0.025

# 95% BI berekenen
ondergrens <- gemiddelde - t_kritiek * se
bovengrens <- gemiddelde + t_kritiek * se

# Resultaat
cat("M =", round(gemiddelde, 3), "\n")
cat("SE =", round(se, 3), "\n")
cat("95% BI = [", round(ondergrens, 3), ",", round(bovengrens, 3), "]\n")

Methode 2: Met t.test()

# Eenvoudigste methode
result <- t.test(data_wave1$W1_Relstress1_Gem)

# 95% BI opvragen
result$conf.int

# Volledige output
result
# Output:
# 95 percent confidence interval:
#  3.318  3.582

Factoren die de Breedte Beïnvloeden

Factor Effect Waarom?
Steekproefgrootte (n) n ↑ → BI smaller SE = SD/√n → meer data = preciezer
Standaarddeviatie (SD) SD ↑ → BI breder Meer variabiliteit = meer onzekerheid
Betrouwbaarheidsniveau 95% → 99% = breder Meer zekerheid = breder interval

Simulatie: Effect van n

n = 30:   95% BI = [3.14, 3.76]  (breedte = 0.62)
n = 150:  95% BI = [3.32, 3.58]  (breedte = 0.26)
n = 500:  95% BI = [3.38, 3.52]  (breedte = 0.14)
n = 2000: 95% BI = [3.41, 3.49]  (breedte = 0.08)

Conclusie: Verviervoudigen van n halveert de breedte van het BI!

Andere Betrouwbaarheidsniveaus

Betrouwbaarheidsniveau Kiezen

  • 90% BI: Minder streng, smaller interval (α = 0.10)
  • 95% BI: Standaard in sociale wetenschappen (α = 0.05)
  • 99% BI: Zeer streng, breder interval (α = 0.01)
# 90% BI
t.test(data_wave1$W1_Relstress1_Gem, conf.level = 0.90)$conf.int

# 95% BI (default)
t.test(data_wave1$W1_Relstress1_Gem)$conf.int

# 99% BI
t.test(data_wave1$W1_Relstress1_Gem, conf.level = 0.99)$conf.int

# Vergelijk breedte
ci_90 <- t.test(data_wave1$W1_Relstress1_Gem, conf.level = 0.90)$conf.int
ci_95 <- t.test(data_wave1$W1_Relstress1_Gem, conf.level = 0.95)$conf.int
ci_99 <- t.test(data_wave1$W1_Relstress1_Gem, conf.level = 0.99)$conf.int

cat("90% BI breedte:", round(diff(ci_90), 3), "\n")
cat("95% BI breedte:", round(diff(ci_95), 3), "\n")
cat("99% BI breedte:", round(diff(ci_99), 3), "\n")

Veelgemaakte Fouten

1. Foutieve Interpretatie van "95%"

❌ Fout: "Er is 95% kans dat de populatiewaarde tussen 3.32 en 3.58 ligt"

✅ Correct: "We zijn 95% confident dat dit interval de populatiewaarde bevat"

2. Verwarring met Standaarddeviatie

❌ Fout: BI zegt iets over de spreiding in de data

✅ Correct: BI zegt iets over de onzekerheid rond het gemiddelde

Voor spreiding gebruik je SD of IQR, niet BI!

3. Overlappende BI = Geen Verschil?

❌ Fout: "De BI overlappen, dus er is geen significant verschil"

✅ Correct: Overlappende BI suggereren geen verschil, maar je moet een t-toets doen voor zekerheid

Praktisch Gebruik

Wanneer Gebruik je BI?

  1. Schatting populatieparameters: "Het gemiddelde inkomen in België ligt tussen €35.000 en €38.000"
  2. Precisie beoordelen: Smal BI = precieze schatting, breed BI = onzekere schatting
  3. Groepen vergelijken: Kijk of BI overlappen (informeel) of doe t-toets (formeel)
  4. Effect sizes rapporteren: Cohen's d met 95% BI geeft effect én onzekerheid

Complete Functie voor Rapportage

rapporteer_met_ci <- function(x, var_name = "Variable", conf_level = 0.95) {
  # Bereken statistieken
  n <- sum(!is.na(x))
  m <- mean(x, na.rm = TRUE)
  sd_val <- sd(x, na.rm = TRUE)
  se <- sd_val / sqrt(n)
  
  # BI berekenen
  ci <- t.test(x, conf.level = conf_level)$conf.int
  
  # Output
  cat("\n=== Beschrijvende Statistieken:", var_name, "===\n\n")
  cat("N:", n, "\n")
  cat("M:", round(m, 2), "\n")
  cat("SD:", round(sd_val, 2), "\n")
  cat("SE:", round(se, 3), "\n")
  cat(paste0(conf_level * 100, "% BI = ["),
      round(ci[1], 2), ",", round(ci[2], 2), "]\n\n")
  
  cat("Interpretatie:\n")
  cat("We zijn", paste0(conf_level * 100, "%"), "confident dat het populatiegemiddelde\n")
  cat("tussen", round(ci[1], 2), "en", round(ci[2], 2), "ligt.\n")
}

# Gebruik
rapporteer_met_ci(data_wave1$W1_Relstress1_Gem,
                  var_name = "Relatiestress",
                  conf_level = 0.95)

Samenvattend

  • 95% BI: Bereik waarbinnen populatieparameter waarschijnlijk ligt
  • Formule: M ± t × SE (waarbij SE = SD/√n)
  • Breder BI: Meer onzekerheid (kleine n, grote SD, hoger betrouwbaarheidsniveau)
  • Smaller BI: Meer precisie (grote n, kleine SD, lager betrouwbaarheidsniveau)
  • Interpretatie: "We zijn 95% confident dat..." (NIET "95% kans dat...")
  • R Code: t.test(x)$conf.int voor snelle berekening
