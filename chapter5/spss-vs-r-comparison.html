üîÑ SPSS vs R: Variabele Manipulatie Vergelijking

Deze gids helpt SPSS-gebruikers om over te stappen naar R door directe vergelijkingen te tonen van veelgebruikte bewerkingen.

Voor SPSS-gebruikers: R kan ALLES wat SPSS kan, en veel meer! De syntaxis is anders, maar de logica is hetzelfde. Deze gids toont je exact hoe.

1Ô∏è‚É£ Nieuwe Variabele Maken (COMPUTE)

üìä SPSS Syntax

EXECUTE.

üìä R Code

data <- data |>
  dplyr::mutate(
    leeftijd_maanden = leeftijd_jaren * 12
  )

Kernverschillen

Kernverschillen tussen SPSS COMPUTE en R mutate
Aspect SPSS R
Commando COMPUTE dplyr::mutate()
Toewijzing = = (binnen mutate)
Afsluiten . (punt) Geen (haakjes sluiten is genoeg)
Uitvoeren EXECUTE. nodig Direct uitgevoerd

2Ô∏è‚É£ Hercoderen naar Zelfde Variabele (RECODE)

üìä SPSS Syntax

RECODE leeftijd (18 thru 25=1) (26 thru 40=2) (41 thru 65=3) (66 thru highest=4).
EXECUTE.

üìä R Code

data <- data |>
  dplyr::mutate(
    leeftijd = dplyr::case_when(
      leeftijd >= 18 & leeftijd <= 25 ~ 1,
      leeftijd >= 26 & leeftijd <= 40 ~ 2,
      leeftijd >= 41 & leeftijd <= 65 ~ 3,
      leeftijd >= 66 ~ 4
    )
  )
Let op: In R overschrijf je de originele variabele als je dezelfde naam gebruikt!
In SPSS gebeurt dit ook met RECODE, maar in R is het explicieter.

3Ô∏è‚É£ Hercoderen naar Nieuwe Variabele (RECODE INTO)

üìä SPSS Syntax

RECODE leeftijd (18 thru 25=1) (26 thru 40=2) (41 thru 65=3) (66 thru highest=4) 
  INTO leeftijd_cat.
EXECUTE.

üìä R Code

data <- data |>
  dplyr::mutate(
    leeftijd_cat = dplyr::case_when(
      leeftijd >= 18 & leeftijd <= 25 ~ 1,
      leeftijd >= 26 & leeftijd <= 40 ~ 2,
      leeftijd >= 41 & leeftijd <= 65 ~ 3,
      leeftijd >= 66 ~ 4
    )
  )
In R: Gebruik gewoon een nieuwe variabelenaam in mutate() - geen apart INTO commando nodig!

4Ô∏è‚É£ Conditionele Bewerkingen (IF)

üìä SPSS Syntax

IF (geslacht = 1) risico_cat = 'Hoog'.
IF (geslacht = 2) risico_cat = 'Laag'.
EXECUTE.

üìä R Code (Methode 1: case_when)

data <- data |>
  dplyr::mutate(
    risico_cat = dplyr::case_when(
      geslacht == 1 ~ "Hoog",
      geslacht == 2 ~ "Laag"
    )
  )

üìä R Code (Methode 2: ifelse)

data <- data |>
  dplyr::mutate(
    risico_cat = ifelse(geslacht == 1, "Hoog", "Laag")
  )

Welke Methode Kiezen?

Wanneer ifelse versus case_when te gebruiken
Situatie Gebruik Reden
2 categorie√´n (ja/nee) ifelse() Korter en simpeler
3+ categorie√´n case_when() Overzichtelijker
Complexe voorwaarden case_when() Makkelijker te lezen

5Ô∏è‚É£ Schaalvariabele Maken (MEAN, SUM)

üìä SPSS Syntax: Mean van Items

  dplyr::mutate(

      na.rm = FALSE

üìä SPSS Syntax: Sum van Items

üìä R Code: Sum van Items

data <- data |>
  dplyr::mutate(
    stress_som = rowSums(
      dplyr::select(., stress1, stress2, stress3, stress4),

Missing Values Afhandeling

SPSS Functie R Equivalent Betekenis
MEAN(item1, item2) rowMeans(..., na.rm=TRUE) Negeer missing, bereken mean van beschikbare
MEAN.2(item1, item2, item3) Custom functie nodig Minimaal 2 geldige waarden vereist
SUM(item1, item2) rowSums(..., na.rm=TRUE) Negeer missing, som van beschikbare
R Voordeel: na.rm = FALSE geeft NA als √©√©n item missing is (striktere controle!).
SPSS berekent vaak mean/sum met missing waarden zonder waarschuwing.

6Ô∏è‚É£ Reverse Coding

Identiek! De formule (Max + Min) - Oude waarde werkt hetzelfde in beide programma's.
Voor 5-puntsschaal: 6 - item (want 5+1=6)

8Ô∏è‚É£ Selecteren van Cases (SELECT IF, FILTER)

üìä R Code: filter()

VERSCHIL:
- SPSS SELECT IF: Verwijdert cases permanent uit dataset!

9Ô∏è‚É£ Groeperen en Aggregeren (AGGREGATE)

üìä SPSS Syntax

  /BREAK=buurt_id
  /gem_inkomen=MEAN(inkomen).

üìä R Code

data <- data |>
  dplyr::group_by(buurt_id) |>
  dplyr::mutate(gem_inkomen = mean(inkomen, na.rm = TRUE)) |>
  dplyr::ungroup()

Alternatief: Aparte Aggregatie Tabel

# Maak aparte tabel met gemiddeldes per buurt
buurt_stats <- data |>
  dplyr::group_by(buurt_id) |>
  dplyr::summarise(gem_inkomen = mean(inkomen, na.rm = TRUE))

# Voeg terug samen met originele data
data <- data |>
  dplyr::left_join(buurt_stats, by = "buurt_id")

Belangrijkste Voordelen van R

‚úÖ Waarom R Beter Is:

  1. Reproduceerbaar: Alle stappen in script (geen klikken in menu's)
  2. Pipe operator: |> maakt stap-voor-stap workflow overzichtelijk
  3. Flexibeler: Oneindig aanpasbaar, niet beperkt tot menu-opties
  4. Gratis: Open source, geen dure licenties
  5. Moderne packages: dplyr, ggplot2, etc. voor effici√´nte analyses
  6. Veilig: Originele data blijft intact (tenzij je expliciet overschrijft)
  7. Community: Enorme online community met hulp en tutorials

Praktisch Voorbeeld: Complete Workflow

Dezelfde Analyse in SPSS en R

üìä SPSS Versie

* Selecteer volwassenen
SELECT IF (leeftijd >= 18).

* Maak leeftijdscategorie
RECODE leeftijd (18 thru 30=1) (31 thru 50=2) (51 thru 99=3) INTO leeftijd_cat.

* Maak stress schaal
COMPUTE stress_gem = MEAN(stress1, stress2, stress3, stress4).

* Maak hoog/laag stress
IF (stress_gem >= 3) stress_niveau = 'Hoog'.
IF (stress_gem < 3) stress_niveau = 'Laag'.

EXECUTE.

* Beschrijvende statistieken
DESCRIPTIVES VARIABLES=stress_gem.

* Kruistabel
CROSSTABS /TABLES=leeftijd_cat BY stress_niveau.

üìä R Versie

# Complete workflow in 1 pipe
data_analyzed <- data |>
  # Selecteer volwassenen
  dplyr::filter(leeftijd >= 18) |>
  
  # Maak leeftijdscategorie
  dplyr::mutate(
    leeftijd_cat = dplyr::case_when(
      leeftijd >= 18 & leeftijd <= 30 ~ 1,
      leeftijd >= 31 & leeftijd <= 50 ~ 2,
      leeftijd >= 51 ~ 3
    )
  ) |>
  
  # Maak stress schaal
  dplyr::mutate(
    stress_gem = rowMeans(
      dplyr::select(., stress1:stress4),
      na.rm = FALSE
    )
  ) |>
  
  # Maak hoog/laag stress
  dplyr::mutate(
    stress_niveau = ifelse(stress_gem >= 3, "Hoog", "Laag")
  )

# Beschrijvende statistieken
summary(data_analyzed$stress_gem)

# Kruistabel
table(data_analyzed$leeftijd_cat, data_analyzed$stress_niveau)
R Voordeel: Alle stappen in √©√©n logische flow met pipe |>!
SPSS vereist meerdere aparte commando's met EXECUTE. tussendoor.
