üîß R Operators: Volledige Naslaggids

In R zijn operators speciale symbolen die bewerkingen uitvoeren op variabelen en waarden. Deze gids behandelt alle operators die je nodig hebt voor variabele manipulatie.

1Ô∏è‚É£ Rekenkundige Operators (Arithmetic)

Voor wiskundige bewerkingen op numerieke variabelen.

Rekenkundige operators in R
Operator Betekenis Voorbeeld Resultaat
+ Optellen 5 + 3 8
- Aftrekken 5 - 3 2
* Vermenigvuldigen 5 * 3 15
/ Delen 6 / 3 2
^ of ** Macht (exponent) 2 ^ 3 8 (2¬≥)
%% Modulo (rest na deling) 7 %% 3 1 (rest)
%/% Integer deling (afgerond) 7 %/% 3 2

Praktijkvoorbeelden

# BMI berekenen
data <- data |>
  dplyr::mutate(
    BMI = gewicht_kg / (lengte_meter ^ 2)
  )

# Leeftijd in jaren naar maanden
data <- data |>
  dplyr::mutate(
    leeftijd_maanden = leeftijd_jaren * 12
  )

# Percentage berekenen
data <- data |>
  dplyr::mutate(
    percentage_correct = (aantal_correct / totaal_vragen) * 100
  )

2Ô∏è‚É£ Vergelijkingsoperators (Relational)

Voor het vergelijken van waarden. Resulteren altijd in TRUE of FALSE.

Vergelijkingsoperators in R
Operator Betekenis Voorbeeld Resultaat
== Gelijk aan 5 == 5 TRUE
!= Niet gelijk aan 5 != 3 TRUE
> Groter dan 5 > 3 TRUE
< Kleiner dan 5 < 3 FALSE
>= Groter dan of gelijk aan 5 >= 5 TRUE
<= Kleiner dan of gelijk aan 5 <= 3 FALSE
%in% Zit waarde in vector? 5 %in% c(1,3,5,7) TRUE
LET OP: Gebruik == (dubbel is-teken) voor vergelijking, NIET = (enkele)!
= wordt gebruikt voor toewijzing (zoals <-)

Praktijkvoorbeelden

# Leeftijdscategorie√´n maken
data <- data |>
  dplyr::mutate(
    leeftijd_cat = dplyr::case_when(
      leeftijd < 18 ~ "Minderjarig",
      leeftijd >= 18 & leeftijd < 65 ~ "Volwassen",
      leeftijd >= 65 ~ "Gepensioneerd"
    )
  )

# Hoog risico identificeren
data <- data |>
  dplyr::mutate(
    hoog_risico = dplyr::case_when(
      risicoscore >= 7 ~ "Ja",
      risicoscore < 7 ~ "Nee"
    )
  )

# Specifieke waarden selecteren
data <- data |>
  dplyr::mutate(
    grote_stad = dplyr::case_when(
      stad %in% c("Antwerpen", "Gent", "Brussel", "Charleroi") ~ "Grote stad",
      TRUE ~ "Andere"
    )
  )

3Ô∏è‚É£ Logische Operators (Logical)

Voor het combineren van meerdere voorwaarden.

Logische operators in R
Operator Betekenis Voorbeeld TRUE wanneer...
& EN (AND) (x > 5) & (x < 10) BEIDE voorwaarden waar
| OF (OR) (x < 5) | (x > 10) MINSTENS √â√âN voorwaarde waar
! NIET (NOT) !(x == 5) Voorwaarde is NIET waar
xor() Exclusieve OF xor(A, B) Precies √â√âN waar (niet beide)

Waarscheidstabellen

Waarscheidstabel voor logische operators
A B A & B (EN) A | B (OF) !A (NIET)
TRUE TRUE TRUE TRUE FALSE
TRUE FALSE FALSE TRUE FALSE
FALSE TRUE FALSE TRUE TRUE
FALSE FALSE FALSE FALSE TRUE

Praktijkvoorbeelden

# Meerdere voorwaarden: BEIDE waar (&)
data <- data |>
  dplyr::mutate(
    jongvolwassen_misdadiger = dplyr::case_when(
      (leeftijd >= 18 & leeftijd < 25) & (aantal_delicten > 0) ~ "Ja",
      TRUE ~ "Nee"
    )
  )

# Meerdere voorwaarden: MINSTENS √â√âN waar (|)
data <- data |>
  dplyr::mutate(
    kwetsbaar = dplyr::case_when(
      (leeftijd < 18) | (inkomen < 1000) | (alleenstaand == "Ja") ~ "Kwetsbaar",
      TRUE ~ "Niet kwetsbaar"
    )
  )

# NIET operator (!)
data <- data |>
  dplyr::mutate(
    geen_slachtoffer = dplyr::case_when(
      !(slachtoffer_misdrijf == "Ja") ~ "Geen slachtoffer",
      TRUE ~ "Wel slachtoffer"
    )
  )

# Complexe combinatie
data <- data |>
  dplyr::mutate(
    risicoprofiel = dplyr::case_when(
      (leeftijd < 25) & (voorstraffen > 2) & (werkloosheid == "Ja") ~ "Hoog risico",
      (leeftijd < 25) & (voorstraffen <= 2) ~ "Gemiddeld risico",
      TRUE ~ "Laag risico"
    )
  )
Tip: Gebruik haakjes () om de volgorde van evaluatie duidelijk te maken!
(A & B) | C is NIET hetzelfde als A & (B | C)

Voor het toewijzen van waarden aan variabelen.

Toewijzingsoperators in R
Operator Betekenis Voorbeeld Gebruik
<- Toewijzing x <- 10 ‚úÖ Voorkeur (R-standaard)
= Toewijzing x = 10 ‚úÖ Binnen functies OK
-> Rechts toewijzen 10 -> x ‚ö†Ô∏è Vermijd (ongebruikelijk)
<<- Globale toewijzing x <<- 10 ‚ö†Ô∏è Gevorderd (functies)
Beste praktijk: Gebruik <- voor toewijzingen aan objecten, en = voor argumenten binnen functies. 
# GOED: <- voor objecten

# NIET AANBEVOLEN: rechts toewijzen

5Ô∏è‚É£ Speciale Operators voor Data Manipulatie

Pipe Operator: |> (native) of %>% (magrittr)

De pipe stuurt het resultaat van de linkerkant door naar de rechterkant als eerste argument.

Pipe operators vergelijking
Operator Package Voorbeeld
|> Base R (vanaf R 4.1) data |> filter(age > 18)
%>% magrittr/dplyr data %>% filter(age > 18) 
# Zonder pipe (genest, moeilijk leesbaar)
result <- select(filter(data, leeftijd > 18), naam, leeftijd)

# Met pipe (stap-voor-stap, leesbaar)
result <- data |>
  dplyr::filter(leeftijd > 18) |>
  dplyr::select(naam, leeftijd)

%in% Operator

Controleert of waarden voorkomen in een vector.

# Selecteer specifieke steden
data <- data |>
  dplyr::filter(stad %in% c("Gent", "Antwerpen", "Brussel"))

# Categoriseer op basis van lijst
data <- data |>
  dplyr::mutate(
    type_misdrijf = dplyr::case_when(
      misdrijf %in% c("diefstal", "inbraak", "beroving") ~ "Vermogenscriminaliteit",
      misdrijf %in% c("aanranding", "moord", "mishandeling") ~ "Geweldscriminaliteit",
      TRUE ~ "Overig"
    )
  )

6Ô∏è‚É£ Operator Volgorde (Precedence)

R evalueert operators in een specifieke volgorde. Van hoogste naar laagste prioriteit:

Operator volgorde (precedence) van hoog naar laag
Prioriteit Operators Betekenis
1 (hoogste) ^ Exponenten
2 - + Unaire min/plus (bijv. -5)
3 : Reeks genereren
4 %any% Speciale operators (%in%, %%, etc.)
5 * / Vermenigvuldigen, delen
6 + - Optellen, aftrekken
7 < > <= >= == != Vergelijkingen
8 ! Logische NIET
9 & Logische EN
10 | Logische OF
11 (laagste) <- = -> Toewijzing

Voorbeelden

# Zonder haakjes (volgt precedence regels)
result <- 2 + 3 * 4        # = 14 (eerst 3*4=12, dan 2+12=14)

# Met haakjes (force volgorde)
result <- (2 + 3) * 4      # = 20 (eerst 2+3=5, dan 5*4=20)

# Complexe expressie
result <- 10 > 5 & 3 < 7   # TRUE (eerst 10>5=TRUE, dan 3<7=TRUE, dan TRUE & TRUE = TRUE)

# Gebruik haakjes voor duidelijkheid
result <- (10 > 5) & (3 < 7)   # Hetzelfde, maar DUIDELIJKER
Beste praktijk: Gebruik altijd haakjes bij twijfel over volgorde!
Leesbaarheid > Korte code
